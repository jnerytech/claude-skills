## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → skill name | User supplies a freeform name via AskUserQuestion or chat; name is used as a filesystem path component |
| skill body → global ~/.claude/skills/ | Skill writes to a global directory using a user-supplied path segment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Skill name → mkdir/Write path in Stage 8 | mitigate | Stage 8 validates name with `^[a-z0-9]+(-[a-z0-9]+)*$` regex AND rejects names containing `/`, `..`, or `\` BEFORE mkdir runs. Validation failure halts — no filesystem side effects. |
| T-03-02 | Tampering | Overwrite of existing skill without user knowledge | mitigate | Stage 7 checks if `$USERPROFILE/.claude/skills/<name>/SKILL.md` already exists; if so, warns user before the "Write it?" gate. User must confirm knowing the overwrite target. |
| T-03-03 | Denial of Service (own write step) | Write called before mkdir creates parent dir | mitigate | Stage 8 enforces strict sequence: validate → `mkdir -p` → Write. Sequence is numbered and explicitly stated in both the Stage 8 instructions and the Final Checks checklist (item 8). |
</threat_model>
