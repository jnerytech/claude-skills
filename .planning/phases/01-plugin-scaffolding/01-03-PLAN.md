<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| docs/ at runtime | skill-create reads docs/ as reference material; content is user-downloaded static files |
| External URLs in index.md | URLs in the table point to Anthropic's official docs; they are reference only, not auto-fetched |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | docs/ folder content | accept | docs/ contains only Claude Code official documentation that is publicly available. No private content, credentials, or PII. The folder is read-only at runtime — skill-create only reads, never writes here. |
| T-03-02 | Tampering | User-downloaded doc files in docs/ | accept | Users populate docs/ manually. Maliciously crafted documentation placed here would be read by skill-create and could influence generated skill output. Risk: low — the user controls what they download; they are the only threat actor with write access to their own docs/ folder. |
| T-03-03 | Information Disclosure | Source URLs in index.md | accept | URLs reference public Anthropic documentation. No authentication tokens, API keys, or private URLs are embedded. |
| T-03-04 | Denial of Service | Empty or missing docs/ folder | accept | If docs/ is empty, skill-create will not find any doc files and will generate a lower-quality skill. This is a UX degradation, not a security threat. The index.md setup instructions clearly tell users to populate docs/ before using /skill-create. |
</threat_model>
